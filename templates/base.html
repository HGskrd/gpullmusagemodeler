Calibration note

Some of the planner's missing realism is already visible in vLLM runtime metrics: running requests, KV-cache usage, prefix-cache hits, prompt tokens, and generation tokens. vLLM's async scheduling and CUDA-graph capture sizing also move real latency and throughput. In practice, the best calibration loop is empirical: fit the planner's knobs against those metrics for a few representative workloads instead of refining the closed-form model in isolation.
